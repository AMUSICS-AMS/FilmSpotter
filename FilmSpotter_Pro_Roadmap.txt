**Version:** 1.0  
**Date:** February 26, 2026

## EXECUTIVE SUMMARY

FilmSpotter Pro is the premium version of FilmSpotter, adding user accounts, project management, onscreen annotations, version control, and professional import/export capabilities. 

**Strategic Approach:** Build AudioReview first (10 weeks), then clone foundation to FilmSpotter Pro (6 weeks) = 16 weeks total for both products vs. 25 weeks if built separately.

## DATABASE STRUCTURE

```javascript
// Firestore structure for FilmSpotter Pro

users/{userId}/
  profile: {
    name: string,
    email: string,
    createdAt: timestamp,
    plan: 'free' | 'pro'
  }
  
  projects/{projectId}/
    name: string,
    createdAt: timestamp,
    lastUpdated: timestamp,
    videoUrl: string,  // Primary video
    
    collaborators/{collaboratorId}/
      userId: string,
      role: 'owner' | 'editor' | 'reviewer' | 'viewer',
      addedAt: timestamp
    
    versions/{versionId}/
      name: string,  // "v1", "v2", "Final Cut"
      videoUrl: string,
      uploadedAt: timestamp,
      uploadedBy: userId,
      status: 'wip' | 'review' | 'approved',
      
      markers/{markerId}/
        time: number,  // seconds
        timecode: string,
        note: string,
        department: string,
        category: string,
        userId: string,
        createdAt: timestamp,
        status: 'wip' | 'review' | 'approved' | 'final',
        
        // Music region data (if applicable)
        isRegion: boolean,
        regionType: 'music' | 'song',
        linkedMarkerId: string,  // ID of matching In/Out marker
        
        // Music metadata (if department === 'music')
        category: 'source' | 'score',
        usage: 'background' | 'feature',
        
        comments/{commentId}/
          userId: string,
          userName: string,
          text: string,
          timestamp: timestamp,
          editedAt: timestamp (optional),
          parentCommentId: string (for threading)
      
      annotations/{annotationId}/
        time: number,
        type: 'circle' | 'box' | 'arrow' | 'freehand' | 'text',
        x: number,  // percentage
        y: number,  // percentage
        width: number (for box),
        height: number (for box),
        radius: number (for circle),
        points: array (for freehand),
        color: string,
        userId: string,
        createdAt: timestamp
```

## KEY DECISIONS MADE

### 1. Music Regions Implementation
**Decision:** Checkbox to link Music In → Music Out  
**Reasoning:** More flexible than auto-linking every 2nd M keypress  
**Implementation:** 
- User presses M for Music In
- Checkbox appears: "Link to Music Out?"
- If checked, next M creates linked Music Out
- Visual connection drawn on timeline

### 2. Multi-Video Tabs
**Decision:** Save for v1.1 (not critical for launch)  
**Reasoning:** 
- Adds significant complexity
- Most users review one video at a time
- Can validate demand in v1.0 before building

### 3. Decouple Mode Default
**Decision:** Default OFF (everyone synced)  
**Reasoning:**
- Most sessions want synchronized playback
- Decoupling is advanced/occasional use case
- User can toggle on when needed

### 4. Import Priority
**Decision:** CSV and EDL together in Phase 5  
**Reasoning:** 
- Both are industry standards
- Relatively simple to implement together
- MIDI can come later if needed

### 5. Platform Support Priority
**Decision:** Box.net in v1.1, not v1.0  
**Reasoning:**
- Dropbox works well and is widely used
- Box.net requires API integration work
- Can validate demand first

### 6. Build Order
**Decision:** AudioReview first, then FilmSpotter Pro  
**Reasoning:**
- 60% code reuse saves 6 weeks
- AudioReview serves broader market
- Get to revenue faster with AudioReview
- Learn from AudioReview before FilmSpotter Pro

## RISKS & MITIGATION

### Technical Risks

**Risk:** Real-time sync conflicts with annotations  
**Mitigation:** Last-write-wins strategy, conflict resolution UI

**Risk:** Video playback issues on different browsers  
**Mitigation:** Extensive cross-browser testing, fallback for older browsers

**Risk:** Firebase costs scaling with users  
**Mitigation:** Optimize queries, implement caching, monitor usage

**Risk:** SVG annotation performance with many drawings  
**Mitigation:** Limit annotations per frame, optimize rendering

### Business Risks

**Risk:** Users don't want to pay $50-100/month  
**Mitigation:** Validate pricing with beta users, offer annual discount

**Risk:** Free version is "good enough"  
**Mitigation:** Make Pro features compelling (annotations, exports, versions)

**Risk:** Competition from Frame.io, Wipster, etc.  
**Mitigation:** Focus on music/composer niche, better pricing, simpler UX

## COMPETITIVE ANALYSIS

### Frame.io
**Strengths:** Video hosting, transcoding, enterprise features  
**Weaknesses:** Expensive ($19-49/user/month), complex, not music-focused  
**Our Advantage:** Simpler, cheaper, music-specific features (regions)

### Wipster
**Strengths:** Good collaboration, approval workflows  
**Weaknesses:** No music-specific features, expensive  
**Our Advantage:** Music regions, better for composers

### Vimeo Review
**Strengths:** Integrated with Vimeo, familiar UI  
**Weaknesses:** Basic features, no music tools  
**Our Advantage:** Professional music features, better exports

### Current FilmSpotter Free
**Strengths:** Free, simple, works well for basic reviews  
**Weaknesses:** No persistence, no annotations, limited exports  
**Our Advantage:** Pro builds on Free's strengths, adds power features
